## 🎯 设计支柱（Design Pillars）

**核心体验目标**（所有设计决策必须服务这些支柱）：

1. **时空操控感**（Time Control Fantasy） - 玩家感觉能操控时间，而非被动接受
2. **资源管理**（Resource Management） - 时间条的获取、储蓄、使用需要策略
3. **爽快感**（Satisfaction） - 激活时间储蓄时，必须有强烈的视听反馈
4. **技能成长**（Skill Mastery） - 玩家能感受到自己"积攒时间条"技术的进步
5. **易学难精**（Easy to Learn, Hard to Master） - 新手能理解"按Q减速"，高手能精准管理时间条

## 🎮 核心玩法循环（Core Gameplay Loop）

### Moment-to-Moment Loop (0-30秒)
```
玩家移动/射击 → 躲避子弹（完美时机）→ 时间条增长 → 
积满时间条 → 按Q激活"时间储蓄" → 慢动作下虐杀敌人 → 
时间条耗尽 → 恢复正常速度 → 多巴胺分泌
```

**详细流程**：
1. **正常游戏** - 玩家移动、射击、躲避
2. **完美躲避** - 子弹接近时完美闪避（0.2秒内按闪避键），时间条+10%
3. **主动积攒** - 玩家刻意"秀操作"来积攒时间条
4. **激活时间储蓄** - 按Q键，消耗时间条，周围一切变慢
5. **慢动作击杀** - 玩家在慢动作下精准射击，感觉像"动作片主角"
6. **时间条耗尽** - 恢复正常速度，玩家感到"意犹未尽"，想再来一次

### Session Loop (5-30分钟)
```
进入房间 → 清除敌人（积攒时间条）→ 收集奖励 → 
遇到精英怪/Boss → 激活时间储蓄（关键战术）→ 
通关/死亡 → 永久进度提升（时间条容量升级）→ "再玩一局"
```

### Long-Term Loop (小时-周)
```
解锁新角色（不同时间条属性）→ 升级时间条容量 → 
挑战更高难度 → 完成成就（"时间管理大师"）→ 
登上排行榜 → 获取赛季奖励 → 等待新内容更新
```

## ⚡ 机制详细规格

### Mechanic 1: 时间储蓄（Time Bank）

```markdown
## Mechanic: 时间储蓄

**Purpose**: 
- 让玩家从"被动接受时间流逝"变为"主动管理时间"
- 创造"时空操控"的幻想（类似《Braid》+《Max Payne》）
- 提供战术深度：何时使用时间储蓄？现在用还是留到Boss战？

**Player Fantasy**: 
- "我能操控时间，我是时间的主宰"
- "在慢动作下，我能精准击杀所有敌人"

**Input**: 
- 完美闪避（子弹距玩家20单位内按闪避键）→ 时间条+10%
- 击杀敌人 → 时间条+5%
- 按Q键 → 激活时间储蓄（消耗时间条）
- 再次按Q键 / 时间条耗尽 → 取消时间储蓄

**Output**: 
- 激活时：游戏世界时间流速 x0.3，玩家动作正常速度
- 时间条视觉反馈（UI进度条 + 屏幕边缘发光）
- 听觉反馈（时间流速变化的音效）

**Success Condition**: 
- 玩家能清晰理解"如何积攒时间条"
- 激活时间储蓄时，玩家感到"强大"和"掌控感"
- 时间条管理成为战术决策（现在用？还是留到Boss战？）

**Failure State**: 
- 时间条获取太慢（玩家感到"无法使用"）
- 时间条消耗太快（玩家感到"不够用"）
- 时间储蓄无反馈（玩家不知道是否激活成功）

**Edge Cases**:
  - 时间条满时，继续积攒 → 溢出无效（可选：溢出部分转化为"时间护盾"）
  - 激活时间储蓄时受伤 → 时间条额外消耗20%（惩罚机制）
  - 多个敌人同时攻击 → 时间储蓄对所有敌人生效（全局效果）
  - Boss战中激活 → Boss动作也变慢（允许，但Boss有时间抗性）

**Tuning Levers**: 
  - 时间条上限（Base: 100%, [PLACEHOLDER]）
  - 完美闪避恢复量（Base: 10%, [PLACEHOLDER]）
  - 击杀敌人恢复量（Base: 5%, [PLACEHOLDER]）
  - 时间条自动恢复速度（Base: 2%/秒, [PLACEHOLDER]）
  - 时间储蓄消耗速度（Base: 10%/秒, [PLACEHOLDER]）
  - 时间流速（激活时）（Base: 0.3x, [PLACEHOLDER]）
  
**Dependencies**: 
  - 玩家输入系统（闪避键检测完美时机）
  - UI系统（时间条进度条、屏幕特效）
  - 音效系统（时间流速变化音效）
  - 敌人AI系统（时间流速变化时，敌人动作同步变慢）
```

### Mechanic 2: 时间条视觉反馈（Time Bar Visuals）

**目的**: 让玩家清晰感知时间条的积累和消耗

**实现方式**：
1. **UI进度条** - 屏幕底部中央，显示时间条当前值
2. **屏幕边缘发光** - 时间条>50%时，屏幕边缘开始发光（蓝色）
3. **时间条满** - 屏幕剧烈闪烁，提示玩家"可以使用了！"
4. **激活时** - 屏幕边缘金色光晕，时间流速变慢的视觉效果

**视觉层级**：
```
时间条 0-30%：淡蓝色进度条，无屏幕特效
时间条 30-60%：蓝色进度条，屏幕边缘微弱发光
时间条 60-90%：亮蓝色进度条，屏幕边缘明显发光
时间条 90-100%：金色进度条，屏幕剧烈闪烁
激活时：金色光晕，世界变慢，子弹轨迹可见
```

**Tuning Levers**:
- 屏幕发光强度（Base: 0.5α, [PLACEHOLDER]）
- 时间条满闪烁频率（Base: 2Hz, [PLACEHOLDER]）
- 激活时屏幕色调（Base: 金色, [PLACEHOLDER]）

### Mechanic 3: 时间储蓄进度系统（Time Bank Progress）

**目的**: 让玩家感受到进步，提供长期动力

**实现方式**：
1. **时间储蓄计数器** - 每局游戏统计激活次数、总慢动作时间
2. **时间储蓄评级** - 每局结束后评级（S/A/B/C）
3. **永久进度** - 总激活次数解锁奖励（新角色、新皮肤、新武器）

**评级标准**：
```
S级：激活次数 > 10次，总慢动作时间 > 30秒，完美闪避率 > 60%
A级：激活次数 > 7次，总慢动作时间 > 20秒，完美闪避率 > 40%
B级：激活次数 > 4次，总慢动作时间 > 10秒，完美闪避率 > 20%
C级：激活次数 < 4次，总慢动作时间 < 10秒，完美闪避率 < 20%
```

**永久进度奖励**：
```
总激活时间储蓄50次 → 解锁新角色"时间法师"（时间条容量+20%）
总激活时间储蓄200次 → 解锁新武器"时间之弓"（慢动作下射速+50%）
总激活时间储蓄500次 → 解锁成就"时间管理大师"
```

### Mechanic 4: Boss战中的时间储蓄（Boss Time Bank Phase）

**目的**: 让Boss战成为时间储蓄系统的高潮时刻

**实现方式**：
1. **Boss血量50%时**，玩家可以激活时间储蓄来创造输出窗口
2. **Boss有时间抗性** - 时间流速只降低到0.5x（而非0.3x）
3. **时间条消耗速度减半** - Boss战时，时间条持续时间x2
4. **时间储蓄阶段** - Boss每损失25%血量，强制进入"时间裂隙"阶段（所有动作变慢5秒）

**Boss战时间储蓄设计**：
```
阶段1（100-70% HP）：Boss动作正常，时间储蓄完全有效（0.3x流速）
阶段2（70-50% HP）：Boss获得时间抗性（0.5x流速），时间条消耗-30%
阶段3（50-30% HP）：时间裂隙阶段！所有动作变慢5秒，玩家可以疯狂输出
阶段4（30-0% HP）：Boss狂暴，时间储蓄无效（时间抗性100%）
```

**Tuning Levers**:
- Boss时间抗性（Base: 50%, [PLACEHOLDER]）
- 时间裂隙持续时间（Base: 5s, [PLACEHOLDER]）
- 时间裂隙冷却时间（Base: 30s, [PLACEHOLDER]）

## 🚨 实现注意事项（For Programmer）

### 技术约束
1. **时间缩放** - 使用`Engine.time_scale`控制全局时间流速
   - 激活时：`Engine.time_scale = 0.3`
   - 玩家动作正常：需要在玩家脚本中忽略时间缩放
   
2. **输入处理** - Q键激活时间储蓄
   - 检测时间条是否>0
   - 激活后，时间条开始消耗（每秒-10%）
   
3. **性能优化** - 时间缩放可能影响物理模拟
   - 使用`delta`时间计算，而非固定帧数
   - 避免时间缩放时的物理异常

### 代码片段参考（Godot 4.x）

```gdscript
## Player.gd - 时间储蓄系统核心逻辑

var time_bank: float = 0.0  # 时间条当前值（0-100）
var time_bank_max: float = 100.0
var time_bank_regen_perfect_dodge: float = 10.0  # 完美闪避恢复量
var time_bank_regen_kill: float = 5.0  # 击杀敌人恢复量
var time_bank_regen_passive: float = 2.0  # 每秒自动恢复量
var time_bank_drain_rate: float = 10.0  # 激活时每秒消耗
var is_time_bank_active: bool = false
var time_slow_scale: float = 0.3  # 时间流速

func _physics_process(delta: float) -> void:
  ## 被动恢复时间条
  if not is_time_bank_active:
    time_bank = min(time_bank + time_bank_regen_passive * delta, time_bank_max)
  
  ## 检测Q键输入
  if Input.is_action_just_pressed("activate_time_bank"):
    _toggle_time_bank()
  
  ## 如果时间储蓄激活，消耗时间条
  if is_time_bank_active:
    time_bank -= time_bank_drain_rate * delta
    if time_bank <= 0:
      _deactivate_time_bank()

func _toggle_time_bank() -> void:
  if is_time_bank_active:
    _deactivate_time_bank()
  else:
    if time_bank > 0:
      _activate_time_bank()

func _activate_time_bank() -> void:
  is_time_bank_active = true
  Engine.time_scale = time_slow_scale
  _play_sound("time_bank_activate")
  _show_visual_feedback("activate")
  
  ## 玩家动作忽略时间缩放
  set_process_input(true)  # 输入正常
  set_physics_process(true)  # 物理正常

func _deactivate_time_bank() -> void:
  is_time_bank_active = false
  Engine.time_scale = 1.0
  _play_sound("time_bank_deactivate")
  _show_visual_feedback("deactivate")

func _on_perfect_dodge() -> void:
  ## 完美闪避回调（由闪避系统调用）
  time_bank = min(time_bank + time_bank_regen_perfect_dodge, time_bank_max)
  _play_sound("time_bank_gain")
  _show_floating_text("时间+10%", Color.CYAN)

func _on_enemy_killed() -> void:
  ## 击杀敌人回调（由敌人系统调用）
  time_bank = min(time_bank + time_bank_regen_kill, time_bank_max)
```

## 📝 Playtest计划

### Playtest目标
1. 验证时间条积攒速度是否合理（完美闪避+10%，击杀+5%）
2. 验证时间储蓄持续时间是否平衡（10%/秒消耗，约10秒持续）
3. 收集玩家反馈（满意度1-10分）

### Playtest流程
```
1. 准备5-10名测试玩家（新手/老手各半）
2. 让玩家玩30分钟（包含教学 + 3-5局游戏）
3. 观察玩家行为（不指导，记录困惑点）
4. 填写问卷：
   - 时间储蓄系统是否有趣？（1-10分）
   - 时间条积攒速度是否太慢/太快？（1-10分）
   - 时间储蓄激活时是否有爽快感？（1-10分）
   - 你是否想继续玩？（是/否）
5. 整理数据，调整参数
```

### 成功标准
- ✅ 时间储蓄系统有趣度 > 7/10
- ✅ 时间条积攒速度适中（新手能激活2-3次/房间，老手5-7次/房间）
- ✅ 时间储蓄爽快感 > 8/10
- ✅ 继续游玩意愿 > 70%

   - 方案A：手动激活（推荐，给予玩家控制权）
