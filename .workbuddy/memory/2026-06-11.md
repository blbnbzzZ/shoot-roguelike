## 虫子Boss修复：间距/移速/黏在一起/头部碰撞伤害

### 修复列表
1. **黏在一起（间距）**：纹理有效内容约 1213×1287 像素，`pixel_size=0.04` 显示宽度约 50 单位。`segment_spacing`: 3.96 → **37.0**（约显示宽度的 75%，每块肉仅有 1/4 重叠）

## 虫子Boss修复：分裂生成完整虫/出生动画/强制转向

### 修复列表
1. **分裂时生成完整虫子**：`_split_at_index()` 中 `add_child(new_worm)` 在 `new_worm._is_split_worm = true` 之前，导致 `_ready()` 执行时标志还是 false，创建了15个完整部位。修复：先设 `_is_split_worm = true` 再 `add_child`
2. **出生动画**：`_create_segments()` 中所有部位初始位置改为 `global_position`（叠在一起），`play_spawn_animation()` 让所有部位一起淡入缩放，然后 `physics_process` 中头部移动，身体通过历史路径自然拉开
3. **强制转向**：`_pick_new_direction()` 改为强制垂直切换（上下→左右，左右→上下），`_reset_direction_timer()` 改为 3-6 秒随机

## 虫子Boss修复：报错/移除走向限制

### 修复列表
1. **修复Parse Error**：`sprite.billboard = GeometryInstance3D.BILLBOARD_DISABLED` → `BaseMaterial3D.BILLBOARD_DISABLED`（`GeometryInstance3D` 无此枚举常量）
2. **移除走向限制**（用户要求不限制，方便后续大地图使用）：
   - 删除 `_spawn_position`、`_boundary_check()`（边界保护）
   - 删除 `_is_wall_ahead()`、`_is_direction_blocked()`（射线检测撞墙）
   - 删除 `_dir_to_vec()`、`_get_all_segment_nodes()`（仅被上述函数使用）
   - 恢复 `_apply_movement()` 为简单版本（纯移动，不检测墙壁）
   - 恢复 `_pick_new_direction()` 为纯随机换方向
3. **保留朝向旋转**：`_update_segment_rotations()` 继续生效（之前因 Parse Error 根本没运行）
3. **移速削弱四分之一**：`move_speed`: 280 → **210**
4. **头部撞玩家没伤害**：WormBoss.tscn 主节点 `CollisionShape3D` 从 `CapsuleShape3D(radius=30, height=60)` 缩小为 `(radius=2, height=4)`。原大碰撞体包裹了头部 HitBox，物理阻挡玩家进入触发区域，导致头部无法造成伤害。缩小后所有部位（包括头部）的 HitBox 都能正常触发。
