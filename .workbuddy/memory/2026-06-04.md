## 修复射击逻辑

**问题**：玩家射击只朝角色面朝方向发射，不向鼠标点击位置射击；长按无法连续射击。

**根本原因**：
1. `_try_fire()` 使用 `_facing_direction`（移动朝向）而非鼠标方向
2. 长按连续射击逻辑缺失（`_physics_process` 中未处理 `_is_firing`）
3. 第一次编辑时语法错误（类级变量前加了制表符），导致脚本加载失败，游戏无任何变化

**修复内容**：
- 新增 `_mouse_world_pos` 和 `_mouse_direction` 变量，存储鼠标在世界坐标中的位置和方向
- 新增 `_update_mouse_world_pos()` 函数，通过摄像机射线与平面 y=0 的交点计算鼠标世界坐标
- 修改 `_try_fire()` 使用 `_mouse_direction` 替代 `_facing_direction`
- 在 `_physics_process` 开头添加每帧调用 `_update_mouse_world_pos()`
- 在 `_physics_process` 中添加长按射击逻辑（`_is_firing && _state != DEAD` 时调用 `_try_fire()`）
- 重写整个 Player.gd 文件，修复第一次编辑引入的语法错误

**关键代码**：
```gdscript
func _update_mouse_world_pos() -> void:
    var mouse_pos: Vector2 = get_viewport().get_mouse_position()
    var ray_origin: Vector3 = cam.project_ray_origin(mouse_pos)
    var ray_dir: Vector3 = cam.project_ray_normal(mouse_pos)
    var t: float = -ray_origin.y / ray_dir.y
    _mouse_world_pos = ray_origin + ray_dir * t
    var to_mouse: Vector3 = _mouse_world_pos - global_position
    _mouse_direction = Vector3(to_mouse.x, 0.0, to_mouse.z).normalized()
```

## 修复碰撞层和子弹检测逻辑

**问题**：玩家打不到敌人，攻击敌人没有任何反应；子弹碰到墙壁不会消失。

**根本原因**：
1. **Godot 碰撞层是位掩码（bitmask）**，之前设置 `collision_layer = 16` 实际是 Layer 5，不是 Layer 16
   - Layer 1 = 1 (2^0), Layer 2 = 2 (2^1), ..., Layer 16 = 32768 (2^15)
2. 子弹的 `_on_body_hit` 函数依赖 `body.is_in_group("enemy")` 检查，可能不可靠
3. 墙壁的 `collision_layer` 设置错误（设为 16 而不是 32768）

**修复内容**：
1. **修正所有碰撞层设置**（使用正确的位掩码值）：
   - 玩家：`collision_layer = 1`, `collision_mask = 32768` (Wall)
   - 敌人：`collision_layer = 2`, `collision_mask = 32768` (Wall)
   - 玩家子弹：`collision_layer = 8`, `collision_mask = 32770` (Enemy + Wall)
   - 敌人子弹：`collision_layer = 32`, `collision_mask = 32769` (Player + Wall)
   - 墙壁：`collision_layer = 32768` (Wall)

2. **优化子弹碰撞检测逻辑**（不再依赖组检查）：
   - 修改 `_on_body_hit()`：直接调用 `_try_damage()`，不检查组
   - 修改 `_try_damage()`：直接查找 `HealthComponent`，如果当前节点没有则查找父节点

**修改的文件**：
- `Player.tscn`：修正 `collision_mask = 32768`
- `EnemyBase.tscn`：修正 `collision_mask = 32768`
- `Projectile.tscn`：修正 `collision_mask = 32770`
- `EnemyProjectile.tscn`：修正 `collision_layer = 32`, `collision_mask = 32769`
- `RoomTemplate.tscn`：所有墙壁的 `collision_layer = 32768`
- `Projectile.gd`：优化 `_on_body_hit()` 和 `_try_damage()` 逻辑
- `EnemyProjectile.gd`：优化 `_on_body_hit()` 和 `_try_damage()` 逻辑

**关键代码**（Projectile.gd 优化后）：
```gdscript
func _on_body_hit(body: Node3D) -> void:
    ## 忽略自己发出的子弹（如果有）
    if body == get_parent():
        return
    ## 尝试对任何有 HealthComponent 的物体造成伤害
    _try_damage(body)
    queue_free()

func _try_damage(target: Node) -> void:
    ## 直接查找 HealthComponent，不依赖组
    var hc := target.get_node_or_null("HealthComponent") as HealthComponent
    if not hc:
        ## 如果 target 本身没有，尝试找父节点
        var parent := target.get_parent()
        if parent:
            hc = parent.get_node_or_null("HealthComponent") as HealthComponent
    if hc and not hc.invincible:
        hc.apply_damage(damage, self)
        enemy_hit.emit()
```

## 修复子弹穿墙/穿敌人（高速穿过和初始重叠检测）

**问题**：玩家子弹打不到敌人，直接穿过去；敌人子弹也穿墙；截图显示信号连接错误。

**根本原因**：
1. **Area3D 的 `body_entered` 信号不可靠**：子弹速度太快时可能在一帧内穿过敌人，不触发信号（tunneling）
2. **子弹在敌人体内生成时不触发 `body_entered`**：近距离射击时子弹生成位置就在敌人碰撞体内，Area3D 不会触发进入事件
3. **`get_overlapping_bodies()` 在 `_physics_process` 中可能返回旧值**：因为物理引擎的重叠检测在 `_physics_process` 调用之后才更新
4. **信号签名不匹配**：`HealthComponent.died(who: Node)` 发射时带参数，但 `Main.gd` 和 `GameManager.gd` 的接收函数没有参数

**修复内容**：
1. **Projectile.gd / EnemyProjectile.gd**：
   - 添加 `_has_hit` 标志防止重复触发
   - 使用 `PhysicsServer3D.intersect_shape()` 主动查询形状重叠（绕过 Area3D 的延迟检测）
   - 在 `_ready()` 中使用 `call_deferred` 延迟检测初始重叠（等物理空间注册完成）
   - 在 `_physics_process` 中每帧主动检测，配合 `force_update_transform()`
   - 修改 `_on_body_hit()` 和 `_try_damage()` 直接查找 `HealthComponent`，不依赖组

2. **信号错误修复**：
   - `Main.gd`：`_on_player_died(_who: Node = null)` 添加默认参数
   - `GameManager.gd`：`_on_player_died(_who: Node = null)` 添加默认参数

**关键代码**（Projectile.gd 主动形状检测）：
```gdscript
func _check_overlaps() -> void:
    if _has_hit:
        return
    var space_state := get_world_3d().direct_space_state
    var shape_query := PhysicsShapeQueryParameters3D.new()
    shape_query.shape = $CollisionShape3D.shape
    shape_query.transform = Transform3D(Basis(), global_position)
    shape_query.collision_mask = collision_mask
    shape_query.exclude = [self]
    var results := space_state.intersect_shape(shape_query, 32)
    for result in results:
        var body := result.collider as Node3D
        if body != get_parent():
            _on_body_hit(body)
            return
```
