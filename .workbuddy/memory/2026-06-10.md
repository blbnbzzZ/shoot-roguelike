### Bug 修复（15:15）- 所有场景文件损坏（致命错误）
- **问题**：大量报错，所有节点都找不到：
  - `Player.tscn:1 - Parse Error: Expected '['`
  - `Main.tscn:14 - Parse Error: [ext_resource] referenced non-existent resource`
  - `Node not found: "Player"`, `Node not found: "CanvasLayer/UI/HealthBar"` 等
- **根因**：之前用 PowerShell `Set-Content` 修改 `Player.tscn` 的 `collision_mask` 时，**添加了 UTF-8 BOM 标记**（`EF BB BF`）
  - Godot 无法解析带 BOM 的 `.tscn` 文件
  - `Player.tscn` 解析失败 → `Main.tscn` 引用失败 → 整个场景加载失败 → 所有 `@onready` 节点为 null
- **修复方案**：
  1. 用 `sed` 批量删除所有 `.tscn` 文件的 UTF-8 BOM：`sed -i '1s/^\xEF\xBB\xBF//'`
  2. 删除 `Player.tscn` 和所有敌人场景中的 `unique_id` 属性
- **重要教训**：
  - **永远不要**用 PowerShell `Set-Content` 修改 Godot 的 `.tscn` 文件（会添加 BOM）
  - 修改 `.tscn` 文件时，使用 `Edit` 工具或 Bash `sed`，避免编码问题
- **提交**：`aa674c3`

### Bug 修复（16:21）- 大房间门传送 bug（蹭门+回到中间）
- **问题1**：大房间与小房间之间的门需要"蹭"很久才能过去
- **问题2**：从小房间返回大房间时，玩家直接出现在大房间中间，而不是从对应门出来
- **原因1**：`_spawn_bullet_barriers()` 使用 `collision_layer = 4`（墙壁层），但玩家 `collision_mask = 7` 包含该层，导致每个门口都有一道永久隐形墙阻挡玩家
- **原因2**：`_enter_room()` 隐藏旧房间时只禁用了 `PlayerDetector`，没禁用门的 `Area3D`，导致旧房间门和新房间门同时触发 `body_entered`，信号竞争
- **修复方案**：
  1. `Room.gd`：子弹阻挡墙 `collision_layer` 从 `4` 改为 `16`（子弹阻挡层），玩家不再被阻挡
  2. `Main.gd _enter_room()`：隐藏旧房间时额外禁用所有门 `Area3D` 的 `monitoring`
  3. `Main.gd _enter_room()`：添加防御性回退，若遍历 `neighbor_indices` 失败，尝试模糊匹配门名
  4. `Room.gd get_spawn_local_position()`：添加防御性模糊匹配，如 `"SouthDoor"` 匹配 `"SouthDoor1"`/`"SouthDoor2"`
- **提交**：`6d23a0d`

### Bug 修复（17:13）- 射击门自己掉血
- **问题**：玩家射击门口时，子弹穿过门口后打到自己，导致自己掉血
- **原因**：子弹墙 `collision_layer = 16`（子弹阻挡层），但 `Projectile.gd` 和 `EnemyProjectile.gd` 的碰撞检测只检查 `body.collision_layer & 4 != 0`（墙壁层），不检查 `& 16`，导致子弹碰到子弹墙时不会被销毁，而是继续飞行穿过门口
- **修复方案**：
  1. `Projectile.gd` `_physics_process()`：将 `body.collision_layer & 4 != 0` 改为 `body.collision_layer & 4 != 0 or body.collision_layer & 16 != 0`
  2. `Projectile.gd` `_on_body_hit()`：同样修复，同时检查墙壁层(4)和子弹阻挡层(16)
  3. `EnemyProjectile.gd` `_on_body_hit()`：同样修复
- **为什么不改回 `collision_layer = 4`**：如果子弹墙使用 `collision_layer = 4`（墙壁层），玩家 `collision_mask = 7` 包含该层，会导致玩家被子弹墙物理阻挡，出现"蹭门"问题（之前已修复）
- **提交**：`40067b9`
